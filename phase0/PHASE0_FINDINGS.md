# Phase 0 — Hand-Annotated Findings

Companion to the auto-generated `outputs/PHASE0_REPORT.md`. This file contains observations from notebook inspection that don't fit neatly in tabular output. Edited by hand; not regenerated by `run_phase0.py`.

**Headline:** `H = 0.6417` → Phase A is weak; paper restructures around non-myopic learned value as primary contribution. See `PHASE0_PREREG.md` §6 for the deterministic decision rule.

---

## 1. Per-feature divergence (the central finding)

| Feature | Mean rate |
|---|---|
| depth | 0.863 |
| ops | 0.825 |
| leaves | 0.817 |
| isolation | 0.858 |
| **composite (primary)** | **0.642** |
| composite_strict | 0.565 |

**Each individual feature is in the "Phase A baseline" zone (≥0.80), but the composite is decisively in the "weak Phase A" zone (0.50–0.80).**

Mechanism: at any single transition, features can disagree on direction. A FACTOR step might decrease depth (collapsing nested structure) while increasing ops (introducing more multiplications). Each feature counts that step as monotone in *its* direction; the z-scored composite captures the *joint* direction and registers a non-decrease whenever any feature spikes.

**Implication for Phase A:** even a 4-feature hand-designed structural heuristic, optimal in any single dimension, fails in 36% of optimal-path transitions. A search using such a heuristic as f(s) will systematically prune productive branches. This is a stronger version of the §2.1 thesis: it's not just rationals; it's quadratics, polynomials, AND any composite of "natural" structural metrics.

## 2. Per-category breakdown

| Category | Composite rate |
|---|---|
| linear | 1.000 |
| rational | 0.867 |
| polynomial | 0.467 |
| **quadratic** | **0.233** |

Two surprises here:

**(a) Quadratics are the worst, not rationals.** The §2.1 motivating example is rational (`(x²−1)/(x−1) = 3`), but in our test set the FACTOR step on `(x+1)*(x−1)` *cancels back* immediately, giving rate 0.667. By contrast, quadratics like `qua03: (x+1)² = 4` go through the full expand→subtract→factor cycle, with the EXPAND step inflating every feature simultaneously. The composite there registers as non-monotone for 2 of 3 transitions.

**(b) Rationals are surprisingly clean.** Of the 5 rational problems, 3 (rat02/rat03/rat05) are "clear-fractions" patterns where each step strictly reduces depth. Only rat01 (the §2.1 case) and rat04 (factor-and-cancel) show the expected non-monotone factor step. So the §2.1 example is *representative of a sub-pattern*, not of all rationals.

## 3. AC-variant fragility (FLAGGED: max σ = 0.50)

Variable rename (`x → y`) flips monotonicity rate from 0 to 1 on `qua01` and `qua02`, both of which are 1-step factor problems. With only one transition, the rate is binary; the z-scoring pool changes when we add renamed variants, shifting the composite values just enough to flip the comparison.

This is a quirk of (a) 1-step traces having no internal information, (b) the global z-score baseline being sensitive to the AC-variant pool composition. It is **not** evidence that the rewrite system itself is fragile under variable rename.

For Phase 1+, the lesson is: the structural-feature scoring needs to be **stable per problem** (z-score against the problem's own trajectory, not the corpus). Or use unscaled features. This was a Phase 0 design choice noted in pre-reg §4 as deliberate.

Add-permute and Mul-permute variants give identical rates to the original, confirming our feature definitions are AC-stable as intended (counts don't depend on argument order).

## 4. Δφ distribution (step-level deltas)

Histogram of `φ(s_{t+1}) − φ(s_t)` (see `outputs/fig_delta_histogram.png`):

- **Δ < 0** (decrease): majority of transitions, as expected
- **Δ > 0** (increase): clustered around quadratic FACTOR/EXPAND steps and polynomial multi-factor steps; max Δ ≈ +1.5 on `qua03` (the EXPAND_LHS step)
- **Δ ≈ 0** (plateau): rare; most transitions move structure either way

The right tail (Δ > 0) is what kills the composite rate. These are the productive complexification steps the §2.1 paper warns about.

## 5. Specific problems worth flagging

- **qua03** (`(x+1)² = 4`): textbook example of expand-then-collapse. Composite goes UP at step 0→1 (expand), DOWN at step 1→2 (subtract), DOWN at step 2→3 (factor). Rate = 2/3.
- **qua04** (complete the square `x²−4x+1 = 0`): step 1→2 ADDS structure to LHS (the +4 for completing-the-square offset). Composite rises sharply, then collapses at the perfect-square factoring step. Classic intermediate complexification.
- **poly02** (`x⁴−5x²+4 = 0`): rate 0.0. Both transitions ADD multiplicative structure (factor as quadratic in x², then factor each as diff of squares). Even though the END is more "solved" (root enumeration), every intermediate step *appears* worse to a structural heuristic.
- **rat01** (`(x²−1)/(x−1) = 3`, the §2.1 case): rate 2/3. FACTOR_NUMERATOR step inflates ops and leaves; CANCEL recovers; SUBTRACT cleans up. Behaves exactly as §2.1 predicts.

## 6. Caveats & limitations (already in pre-reg, restated)

- Hand-curated traces are not provably BFS-optimal. Bias direction is *toward* monotonicity (longer paths give more decreasing pairs available), so a fail at H=0.64 is a real fail.
- The qua05 trace (`(x−1)*(x−2) = x−1`) deliberately uses the textbook expand-and-refactor 4-step path. A shorter 2-step path (factor-out + simplify) exists. We chose the longer textbook path because it's representative of how a hand-heuristic + beam search would explore *if* it took that branch — and we want to measure exactly that.
- 20 problems is small. No bootstrap CI is reported. The ±0.02 tie-breaker buffer is the only nod to uncertainty.
- AC-fragility on 1-step problems is a z-score-pool artifact, not a finding about the rewrite system. Discount it.

## 7. What this means for the GGMR paper framing

**Decision is firm: paper restructures around non-myopic learned value as the primary contribution.**

This is not just §2.1's "rationals can be non-monotone." It's a stronger result:

> Even composite hand-designed structural heuristics, computed over the four features explicitly mentioned in §4.1 (depth, operator count, term count, variable isolation), are non-monotone on **36% of optimal-path transitions across 20 textbook-canonical problems** — concentrated in **quadratics (77% non-monotone)** and **polynomials (53%)**.

§4.1's hand-designed beam-search baseline will fail on these problems unless given a search horizon long enough to traverse the complexification valleys. A Phase B/C learned non-myopic value function targets exactly this pathology.

**Concrete framing edits to consider:**
1. Promote §2.1's "value monotonicity problem" from motivation to **central claim**.
2. Reframe §4.1 (Phase A) as: "Phase A is included as a foil — we report its performance to quantify the gap a non-myopic value function fills."
3. The four-feature composite from this Phase 0 can serve directly as Phase A's hand-heuristic. Its measured H = 0.64 sets a floor on what beam search can reasonably achieve.
4. Add Phase 0 results to §5.6 ("Prerequisite Validation") in the paper itself — they are themselves a contribution: a quantitative characterization of the value-monotonicity problem on a fixed test set.
